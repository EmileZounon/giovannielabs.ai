Deploiement n8n · 45 minutes

Deployer et tester le bot, etape par etape.

Suivez ces 6 phases dans l'ordre. En 45 minutes chrono, votre bot ecoute le groupe Telegram, detecte les etudiants silencieux et envoie des messages personnalises ecrits par Claude.

Niveau : debutant accompagneDuree : ~45 minCout : ~5 $/mois

Sommaire

  1. Preparation des comptes
  2. Importer le workflow
  3. Tester la capture
  4. Tester les rappels
  5. Tester les cas limites
  6. Activer le cron
Phase 0 · 20 minutes

Preparer les comptes et les cles

Faites ces 6 choses dans l'ordre. Notez chaque token et ID dans un fichier texte temporaire, vous en aurez besoin a la phase 1.

  1. Creer le bot Telegram

    Sur Telegram, cherchez @BotFather et envoyez-lui /newbot. Donnez un nom et un username (doit finir par bot). Copiez le token qu'il vous renvoie.

    Tres important : envoyez ensuite /setprivacy → choisissez votre bot → Disable. Sans ca, le bot ne verra que les commandes, pas les messages normaux.

  2. Creer un groupe Telegram de test

    Creez un nouveau groupe Telegram "Test KodjoBot" avec vous + un ami. Ajoutez le bot comme administrateur du groupe (Parametres → Administrateurs → Ajouter). Envoyez un message test pour verifier.

  3. Creer le Google Sheet

    Creez un nouveau Google Sheet nomme kodjo-engagement-test avec 3 onglets nommes exactement : students, messages, outreach_log.

    Importez les 3 templates CSV pour avoir les en-tetes prets :

    students.csv messages.csv outreach_log.csv

    Copiez l'ID du Sheet depuis l'URL : docs.google.com/spreadsheets/d/ICI_C_EST_L_ID/edit

  4. Recuperer votre propre telegram_user_id

    Sur Telegram, cherchez @userinfobot et envoyez-lui n'importe quel message. Il vous retourne votre user id (un numero comme 123456789). Copiez-le : il servira a vous ajouter comme eleve de test.

  5. Recuperer une cle API Anthropic

    Allez sur console.anthropic.com → API Keys → Create Key. Copiez la cle immediatement, elle ne s'affichera qu'une fois. Elle commence par sk-ant-....

  6. Preparer un compte Gmail

    Le bot enverra les emails fallback depuis ce compte. Utilisez un compte dedie si possible (ex. bot@kodjoenglish.com). Activez la 2FA et creez un mot de passe d'application si necessaire.

    Recapitulatif : notez dans un fichier temporaire vos 4 valeurs : TELEGRAM_BOT_TOKEN, GOOGLE_SHEET_ID, MON_TELEGRAM_USER_ID, ANTHROPIC_API_KEY.
Phase 1 · 10 minutes

Importer le workflow et brancher les credentials

Le fichier JSON contient les 12 nodes deja relies. Il ne reste qu'a connecter les comptes.

  1. Importer le fichier dans n8n Cloud

    Ouvrez n8n Cloud → Workflows → menu "..." en haut a droite → "Import from File". Choisissez kodjo-engagement-bot.json. Le workflow s'ouvre avec 12 nodes visibles.

  2. Creer le credential Telegram

    Cliquez sur Telegram: on group message → choisir Create New Credential → collez votre TELEGRAM_BOT_TOKEN. Enregistrez. Ce meme credential servira pour le node Telegram: send DM.

  3. Creer le credential Google Sheets

    Cliquez sur n'importe lequel des 4 nodes Google Sheets → Create New Credential → choisissez OAuth2 → Sign in with Google. Acceptez les permissions. Ce credential sera automatiquement partage par les 4 nodes.

  4. Creer le credential Anthropic (Header Auth)

    Le node Claude: generate nudge est un HTTP Request. Il utilise un credential de type Header Auth, pas un credential Anthropic dedie.

    • Cliquez sur le node Claude: generate nudge.
    • Dans Authentication, choisissez Generic Credential Type.
    • Dans Generic Auth Type, choisissez Header Auth.
    • Cliquez Create New Credential :
      • Name : Anthropic API Key (nom libre)
      • Header Name : x-api-key (exactement)
      • Header Value : votre cle sk-ant-...
    • Enregistrez. Le node affiche maintenant le credential en vert.
    Pourquoi Header Auth et pas "Anthropic" ? Certaines versions de n8n Cloud n'ont pas le node Anthropic natif. HTTP Request + Header Auth fonctionne partout, sans dependance.

  5. Creer le credential Gmail

    Cliquez sur Gmail: send email → Create New Credential → OAuth2 → Sign in with Google. Utilisez le compte Gmail prepare a la phase 0.

  6. Remplacer l'ID du Google Sheet

    Dans chacun des 4 nodes Google Sheets (log message, Read students, Read messages, Log outreach), remplacez REPLACE_WITH_GOOGLE_SHEET_ID par votre vrai Sheet ID. Selectionnez le bon onglet dans le menu deroulant.

  7. Sauvegarder

    Cliquez sur l'icone disquette en haut a droite. Tous les triangles rouges des nodes doivent avoir disparu.

Phase 2 · 5 minutes

Tester le Flow 1 (capture des messages)

Objectif : verifier que le bot ecoute bien le groupe et remplit l'onglet messages.

  1. Activer le workflow

    Basculez le toggle en haut a droite sur Active (vert). Le Telegram Trigger devient operationnel et ecoute votre groupe.

  2. Envoyer un message test

    Dans votre groupe Telegram de test, envoyez le message ceci est un test.

  3. Verifier dans n8n

    Revenez dans n8n → onglet Executions dans la barre de gauche. Vous devez voir une nouvelle execution avec un statut vert Success.

  4. Verifier dans Google Sheets

    Ouvrez votre Sheet → onglet messages. Une nouvelle ligne doit apparaitre avec votre message et votre telegram_user_id.

Si ca ne marche pas :
  • Verifiez que le toggle est bien sur Active.
  • Verifiez que privacy mode est bien Disable dans BotFather.
  • Verifiez que le bot est admin du groupe.
  • Ouvrez le node "Telegram: on group message" → Test with trigger data.
Phase 3 · 10 minutes

Tester le Flow 2 (rappels aux silencieux)

Objectif : simuler un silence de 4 jours et verifier que Claude genere un message doux, qui part bien en Telegram DM puis en email (fallback).

  1. Vous ajouter comme eleve de test

    Dans l'onglet students, ajoutez une ligne avec vos vraies infos :

    telegram_user_idfirst_nameemailactive
    VOTRE_IDEmilevotre@email.comTRUE

  2. Falsifier un vieux message

    Dans l'onglet messages, ajoutez une ligne avec votre user_id et un timestamp dans le passe, par exemple 4 jours avant aujourd'hui. Ca fait croire au bot que vous n'avez pas ecrit depuis 4 jours.

    Astuce : dans Google Sheets, tapez =NOW()-4 dans la cellule timestamp pour obtenir une date de 4 jours en arriere, puis copiez/collez la valeur.

  3. Faire /start au bot en DM

    Ouvrez une conversation directe avec votre bot (cherchez son username dans Telegram). Envoyez /start. Le bot ne repondra pas, mais il "existe" maintenant dans vos chats. Sans ca, le DM echouera et le fallback email prendra le relais (ce qu'on teste aussi).

  4. Executer le Flow 2 manuellement

    Dans n8n, ouvrez le workflow. Cliquez sur le node Daily at 09:00 puis sur le bouton Execute Workflow (Play). La chaine complete s'execute. Regardez chaque node s'allumer en vert.

  5. Verifier ce que vous recevez

    • Telegram : un DM de votre bot, style "Emile, on pense a toi..."
    • Si vous n'avez pas fait /start : un email a la place, meme contenu.
    • Onglet outreach_log : une nouvelle ligne avec le message envoye, le canal utilise, et le nombre de jours de silence.
Phase 4 · 10 minutes

Tester les cas limites

Trois scenarios qui peuvent casser en prod si on ne les verifie pas.

  1. Fallback email (eleve sans /start)

    Ajoutez un deuxieme eleve de test avec un telegram_user_id valide mais qui n'a jamais fait /start. Ajoutez un vieux message pour lui. Executez le Flow 2.

    Attendu : le Telegram DM echoue → le node Gmail prend automatiquement le relais → l'email part.

  2. Desactivation d'un eleve

    Pour un eleve de test, mettez la colonne active a FALSE. Executez le Flow 2.

    Attendu : aucun message ne part pour cet eleve. Utile pour les periodes de vacances ou de pause volontaire.

  3. Eleve sans Telegram

    Ajoutez un eleve sans telegram_user_id. Notez qu'il ne sera jamais detecte comme silencieux : la v1 ne suit l'engagement que via Telegram. La v2 ajoutera une detection multi-canal.

Phase 5 · 1 minute

Activer le cron quotidien

Une fois tous les tests passes, le workflow est deja actif et le cron se declenchera automatiquement demain matin. Une derniere verification.

  1. Verifier le cron

    Dans le node Daily at 09:00, verifiez que l'expression cron est bien 0 9 * * * (tous les jours a 9h).

  2. Ajuster le fuseau horaire

    Cliquez sur l'icone engrenage en haut du workflow → Workflow settings → Timezone → choisissez Africa/Abidjan (ou le fuseau de Kodjo).

  3. Nettoyer les donnees de test

    Supprimez vous-meme de students, supprimez les vieux messages fake de messages, videz outreach_log. Le systeme est pret pour les vrais eleves.

Checklist finale

Avant de montrer le bot a Kodjo

Tout doit etre coche
  • Flow 1 capte les messages en temps reel dans messages
  • Flow 2 detecte les silencieux au-dela de 48h
  • Claude genere un message dans le ton voulu (doux, encourageant)
  • Telegram DM fonctionne pour les eleves qui ont fait /start
  • Fallback email fonctionne pour les autres
  • Onglet outreach_log se remplit correctement
  • Le cron est configure au bon fuseau horaire
  • Les donnees de test sont nettoyees

Apres la validation

Passage en production

  1. Nettoyer le Sheet de test

    Supprimez toutes les lignes de test dans les 3 onglets.

  2. Dupliquer le workflow en version prod

    Dans n8n : Duplicate workflow → renommer en kodjo-engagement-PROD. Gardez la version test au cas ou.

  3. Ajouter les vrais eleves dans Sheets

    Demandez a Kodjo de remplir l'onglet students avec ses vrais eleves : telegram_user_id, email, active=TRUE.

  4. Ajouter le bot au vrai groupe

    Dans le vrai groupe KodjoEnglish, ajoutez le bot comme admin. Postez le message de consentement (voir la page principale du kit).

  5. Laisser tourner 2-3 semaines

    Avant d'ajouter WhatsApp en v2, observez ce qui marche : combien d'eleves sont touches, combien reviennent dans le groupe apres un rappel, quels messages fonctionnent le mieux. Le outreach_log est votre tableau de bord.
