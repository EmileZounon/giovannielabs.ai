What it does

The bot in 4 steps

01

Listens to the group

Every time a student writes in the Telegram group, the bot records who, when, and what into a Google Sheet. Silent, in the background.

02

Checks at 9 AM

Every morning, it scans the student list and flags anyone who hasn't written for 48 hours or more.

03

Claude writes the message

For each silent student, Claude generates a short personalized note in French, in a warm and caring tone. Two sentences, their first name, a light question.

04

Sends via cascade

Tries Telegram DM first (free, personal). If the student hasn't activated the bot, automatically falls back to email. Every send is logged to your dashboard.

Architecture at a glance

Telegram group | v [Flow 1] Bot listens -> Google Sheet ("messages" tab) [Flow 2] Every day at 9 AM: Google Sheet "students" + "messages" | v Code: who has been silent > 48h? | v Claude Sonnet 4.5: generates the message | v Telegram DM ------(on failure, auto fallback)-----+ | | v v Log to "outreach_log" Email via Gmail | v Log to "outreach_log"

Get started

The 7 deployment steps

Follow them in order — skip one and the rest won't work.

  1. Create the Telegram bot

    On Telegram, find @BotFather. Send /newbot, give a name and username. Copy the token. Critical: /setprivacy → Disable (otherwise the bot won't see regular messages).

  2. Add the bot to the group as admin

    Group settings → Administrators → Add the bot. Without admin status, privacy mode off isn't enough.

  3. Create the Google Sheet

    New Sheet, 3 tabs named exactly students, messages, outreach_log. Import the CSV templates above to get the headers right. Copy the Sheet ID from the URL.

  4. Get an Anthropic API key

    Go to console.anthropic.com → API Keys → Create Key. Copy it and keep it safe.

  5. Import the workflow into n8n

    n8n Cloud → Workflows → Import from File → select kodjo-engagement-bot.json. The workflow opens with 12 visible nodes.

  6. Connect the credentials (4)

    On each relevant node: Telegram (bot token), Google Sheets (OAuth), Gmail (OAuth), Anthropic (Header Auth with x-api-key). In all 4 Google Sheets nodes, replace REPLACE_WITH_GOOGLE_SHEET_ID with your actual ID.

  7. Activate the workflow and test

    Toggle Active in the top right. Follow the testing guide to validate both flows before letting it run.

Estimated cost: n8n Cloud (already paid) + approximately $5/month of Claude API usage for a group of 20 active students. Email is free via Gmail. Telegram is free.
Important notes

What to know before launching

Student consent

Before activating the bot, post this message in the group:

To better support you, our bot tracks your participation in the group. If you haven't written for 48 hours, you'll receive a friendly DM or email. It's just to check in. Reply STOP anytime to opt out of these reminders.

Telegram DM activation

Telegram has a rule: a bot can only DM a student if that student has clicked /start on the bot at least once. Students who haven't will receive their messages via email (automatic fallback).

WhatsApp coming later (v2)

WhatsApp isn't included in v1 because the Meta API requires a business verification that takes about 5 business days. Once v1 is running and validated, WhatsApp slots in as an additional node in the cascade, with no architecture rework.

Message privacy

The bot records the text of every group message into a private Google Sheet. Only the Sheet owner's Google account has access. Delete old rows regularly if you want to limit stored history.
