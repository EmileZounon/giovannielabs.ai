Phase 0 · 20 minutes

Prepare accounts and API keys

Do these 6 things in order. Write every token and ID into a temporary text file — you'll need them in phase 1.

  1. Create the Telegram bot

    On Telegram, find @BotFather and send /newbot. Give it a name and a username (must end in bot). Copy the token it returns.

    Critical: then send /setprivacy → select your bot → Disable. Without this, the bot will only see commands, not regular messages.

  2. Create a test Telegram group

    Create a new Telegram group "Test KodjoBot" with yourself + a friend. Add the bot as an administrator (Settings → Administrators → Add). Send a test message to verify.

  3. Create the Google Sheet

    Create a new Google Sheet named kodjo-engagement-test with 3 tabs named exactly: students, messages, outreach_log.

    Import the 3 CSV templates to get the headers right:

    students.csv messages.csv outreach_log.csv

    Copy the Sheet ID from the URL: docs.google.com/spreadsheets/d/THIS_IS_THE_ID/edit

  4. Get your own telegram_user_id

    On Telegram, find @userinfobot and send it any message. It returns your user id (a number like 123456789). Copy it — you'll use it to add yourself as a test student.

  5. Get an Anthropic API key

    Go to console.anthropic.com → API Keys → Create Key. Copy the key immediately, it's only shown once. It starts with sk-ant-....

  6. Prepare a Gmail account

    The bot will send fallback emails from this account. Use a dedicated one if possible (e.g., bot@kodjoenglish.com). Enable 2FA and create an app password if needed.

    Recap: save these 4 values in your temp file — TELEGRAM_BOT_TOKEN, GOOGLE_SHEET_ID, MY_TELEGRAM_USER_ID, ANTHROPIC_API_KEY.
Phase 1 · 10 minutes

Import the workflow and connect credentials

The JSON file contains the 12 nodes already wired together. All that's left is connecting the accounts.

  1. Import the file into n8n Cloud

    Open n8n Cloud → Workflows → "..." menu top-right → "Import from File". Choose kodjo-engagement-bot.json. The workflow opens with 12 nodes visible.

  2. Create the Telegram credential

    Click on Telegram: on group message → choose Create New Credential → paste your TELEGRAM_BOT_TOKEN. Save. This same credential will be reused by the Telegram: send DM node.

  3. Create the Google Sheets credential

    Click any of the 4 Google Sheets nodes → Create New Credential → choose OAuth2 → Sign in with Google. Accept the permissions. This credential is automatically shared across all 4 nodes.

  4. Create the Anthropic credential (Header Auth)

    The Claude: generate nudge node is an HTTP Request. It uses a Header Auth credential, not a dedicated Anthropic credential.

    • Click on the Claude: generate nudge node.
    • Under Authentication, choose Generic Credential Type.
    • Under Generic Auth Type, choose Header Auth.
    • Click Create New Credential:
      • Name: Anthropic API Key (any label)
      • Header Name: x-api-key (exactly)
      • Header Value: your sk-ant-... key
    • Save. The node now shows the credential in green.
    Why Header Auth and not "Anthropic"? Some versions of n8n Cloud don't include the native Anthropic node. HTTP Request + Header Auth works everywhere, no dependency.

  5. Create the Gmail credential

    Click on Gmail: send email → Create New Credential → OAuth2 → Sign in with Google. Use the Gmail account you prepared in phase 0.

  6. Replace the Google Sheet ID

    In each of the 4 Google Sheets nodes (log message, Read students, Read messages, Log outreach), replace REPLACE_WITH_GOOGLE_SHEET_ID with your actual Sheet ID. Select the correct tab from the dropdown.

  7. Save

    Click the disk icon in the top right. All red warning triangles on the nodes should disappear.

Phase 2 · 5 minutes

Test Flow 1 (message capture)

Goal: verify the bot listens to the group and fills the messages tab.

  1. Activate the workflow

    Flip the toggle in the top right to Active (green). The Telegram Trigger goes live and listens to your group.

  2. Send a test message

    In your test Telegram group, send the message this is a test.

  3. Verify in n8n

    Back in n8n → Executions tab in the left sidebar. You should see a new execution with a green Success status.

  4. Verify in Google Sheets

    Open your Sheet → messages tab. A new row should appear with your message and your telegram_user_id.

If it doesn't work:
  • Check that the toggle is Active.
  • Check that privacy mode is Disable in BotFather.
  • Check that the bot is an admin of the group.
  • Open the "Telegram: on group message" node → Test with trigger data.
Phase 3 · 10 minutes

Test Flow 2 (nudge silent students)

Goal: simulate a 4-day silence and verify that Claude generates a soft message that goes out via Telegram DM, then falls back to email.

  1. Add yourself as a test student

    In the students tab, add a row with your real info:

    telegram_user_idfirst_nameemailactive
    YOUR_IDEmileyour@email.comTRUE

  2. Fake an old message

    In the messages tab, add a row with your user_id and a past timestamp, for example 4 days before today. This tricks the bot into thinking you haven't written in 4 days.

    Tip: in Google Sheets, type =NOW()-4 in the timestamp cell to get a date 4 days in the past, then copy/paste the value.

  3. Send /start to the bot in DM

    Open a direct chat with your bot (search its username in Telegram). Send /start. The bot won't reply, but it now "exists" in your chats. Without this step, the DM will fail and the email fallback will kick in (which is also what we want to test).

  4. Execute Flow 2 manually

    In n8n, open the workflow. Click the Daily at 09:00 node, then click Execute Workflow (Play). The whole chain runs. Watch each node turn green.

  5. Verify what you receive

    • Telegram: a DM from your bot, something like "Emile, we've been thinking about you..."
    • If you didn't /start: an email instead with the same content.
    • outreach_log tab: a new row with the message sent, the channel used, and days silent.
Phase 4 · 10 minutes

Test edge cases

Three scenarios that can break in production if you don't verify them.

  1. Email fallback (student without /start)

    Add a second test student with a valid telegram_user_id but who has never done /start. Add an old message for them. Execute Flow 2.

    Expected: Telegram DM fails → the Gmail node automatically takes over → email goes out.

  2. Student deactivation

    For a test student, set the active column to FALSE. Execute Flow 2.

    Expected: no message goes out for this student. Useful for vacation or intentional pauses.

  3. Student without Telegram

    Add a student without a telegram_user_id. Note they'll never be flagged as silent: v1 tracks engagement only via Telegram. V2 will add multi-channel detection.

Phase 5 · 1 minute

Activate the daily cron

Once all tests pass, the workflow is already active and the cron will trigger automatically tomorrow morning. One last check.

  1. Verify the cron

    In the Daily at 09:00 node, verify the cron expression is 0 9 * * * (every day at 9 AM).

  2. Set the timezone

    Click the gear icon at the top of the workflow → Workflow settings → Timezone → choose Africa/Abidjan (or Kodjo's timezone).

  3. Clean up test data

    Delete yourself from students, delete the fake old messages in messages, empty outreach_log. The system is ready for real students.

Final checklist

Before showing the bot to Kodjo

Everything must be checked
  • Flow 1 captures messages in real time into messages
  • Flow 2 detects silent students beyond 48h
  • Claude generates messages in the right tone (soft, encouraging)
  • Telegram DM works for students who did /start
  • Email fallback works for the others
  • The outreach_log tab fills correctly
  • The cron is set to the right timezone
  • Test data has been cleaned up

After validation

Moving to production

  1. Clean up the test Sheet

    Delete all test rows from the 3 tabs.

  2. Duplicate the workflow as a prod version

    In n8n: Duplicate workflow → rename to kodjo-engagement-PROD. Keep the test version as a backup.

  3. Add real students to Sheets

    Ask Kodjo to fill the students tab with his real learners: telegram_user_id, email, active=TRUE.

  4. Add the bot to the real group

    In the real KodjoEnglish group, add the bot as admin. Post the consent message (see the kit's main page).

  5. Let it run for 2-3 weeks

    Before adding WhatsApp in v2, observe what works: how many students are reached, how many come back after a nudge, which messages perform best. The outreach_log is your dashboard.
